# 測試案例清單 — momo 首頁搜尋

對象：[momo 購物網](https://www.momoshop.com.tw/) 首頁搜尋區，分為**輸入框**、**搜尋按鈕**、**自動完成下拉**三部分。

範圍說明：本清單以行為明確、可直接斷言的情境為主——除正向流程外，另納入兩個行為已確認的邊界 / 負向案例（查無結果、前後空白）。其餘規則仍不明確的情境（空字串送出、特殊字元上限、長度上限等）暫不納入。

優先級：P0 = 核心必測，P1 = 重要，P2 = 加分。

## 搜尋結果頁 — 排序與篩選

> **重要前提（廣告排除）**：搜尋結果前數列為廣告商品，會固定置頂、不隨排序或篩選改變。廣告卡片以 `.sponsor-tag`（顯示「Ad」）辨識——注意廣告卡**同樣帶有評分 / 評價數**，因此不能用「有無評分」判斷，唯一可靠依據是 `.sponsor-tag`。以下排序與價格篩選的斷言**僅針對非廣告的一般商品卡片**，需先過濾掉廣告卡片再驗證。
>
> 價格取值：由卡片價格文字去除 `$` 與千分位逗號後轉為數值再比較。
>
> 星等取值：momo **實心 / 半 / 空心星都會渲染**，且實心與空心星**共用同一 path 前綴、同色 fill**——差別在空心星的 path 多一段「內凹子路徑」把中心挖空成輪廓。故實心 vs 空心需以**子路徑（moveto）段數**判斷（實心 1 段、空心 2 段），半星為獨立 path（前綴 `M21.151`）；**評分 = 實心星數 + 0.5 × 半星數**。（注意：只比對前綴或 fill 會把空心星誤判為滿星。）

### D. 排序 Sort

| ID | 優先 | 測試案例 | 預期結果 |
|----|------|----------|----------|
| SO-01 | P0 | 點「價格」排序，UI 標示高→低（class `down`）時 | 非廣告商品價格呈遞減排列 |
| SO-02 | P1 | 點「價格」排序，UI 標示低→高（class `up`）時 | 非廣告商品價格呈遞增排列 |
| SO-03 | P1 | 點「評價」排序，UI 標示高→低（class `down`）時 | 非廣告商品**星等**呈遞減排列（momo 評價排序鍵為星等，非評價數量） |
| SO-04 | P1 | 點「評價」排序，UI 標示低→高（class `up`）時 | 非廣告商品**星等**呈遞增排列 |

> **排序方向以 UI 為準**：價格與評價排序採同一套處理——方向不以「點第幾次」推斷，而是讀取排序 li 的 class（`down`=高→低、`up`=低→高）；點擊排序直到 UI 顯示目標方向，再驗證實際資料順序與 UI 標示一致。
